
Цей проєкт реалізує локальний ETL (Extract, Transform, Load) пайплайн, який обробляє сирі CSV-дані, очищує та валідує їх, а потім завантажує у PostgreSQL базу даних для подальшого аналізу.

---

## Функціональність

### Extract (Витяг даних)
- Читання сирих CSV файлів:
  - customers.csv
  - products.csv
  - orders.csv
  - order_items.csv

---

### Transform (Очищення даних)
- Видалення дублікатів первинних ключів
- Обробка відсутніх або некоректних ID
- Фільтрація некоректних email-адрес
- Перетворення та перевірка timestamp-ів
- Видалення від’ємних або нульових значень (ціна, кількість)
- Нормалізація статусів замовлень (нижній регістр + фільтрація допустимих значень)
- Обробка відсутніх зовнішніх ключів

---

### Load (Завантаження даних)
- Завантаження очищених даних у PostgreSQL:
  - customers_clean
  - products_clean
  - orders_clean
  - order_items_clean

---

### Перевірка якості даних
- Відсутність дублікатів ID
- Відсутність NULL у критичних полях
- Перевірка цілісності зв’язків між таблицями
- Перевірка допустимих значень статусів
